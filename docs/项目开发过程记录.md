# 项目开发过程记录

本文档用于持续记录本项目从当前雏形到完善上线的开发、优化和决策过程。新 agent 接手时应优先阅读本文件、`AGENTS.md` 和 `.codex/project-context/`。

## 当前接手摘要

- 当前项目是 AI 驱动的电商/营销视觉资产制作 SaaS。
- 技术栈：Next.js 14 App Router、React 18、TypeScript、Prisma、NextAuth、Fabric.js、Zustand、Tailwind CSS。
- 当前处于基础可维护性修复完成、Dashboard 顶部导航和仪表盘背景视觉调整阶段。
- 近期用户明确要求：小 UI 调整后不需要自动跑 `lint/build`，由用户本地检查。除非用户明确要求验证，后续 agent 不要主动运行检查命令。

当前背景实现要求和状态：

注意：用户对背景的最终期望是“红框区域全部由图片铺满，内容滚动但背景不滚动”。如果后续视觉仍不符合，需要优先检查 `layout.tsx` 中 `main` 和内容滚动容器的高度、overflow、z-index。

### 2026-05-06 ~ 2026-05-11：首页 AI 生成链路落地 + 家居家纺场景化

**目标**

- 修复首页输入框无法输入的问题，打通从用户描述到图片生成的完整链路。
- 实现反向提示词优化：用户输入中文自然语言（可附参考图），由 AI 自动优化为适合图像生成模型的高质量提示词。
- 接入真实图片生成 API，替换原有 mock 接口。
- 增加模型选择功能，支持 Stable Image Ultra 和 GPT Image 2 两个模型。
- 将整体产品场景聚焦为家居家纺图案设计，优化 System Prompt 使生成结果贴近商业家纺印花需求。

**改动**

环境与配置：
- 发现系统默认 Node.js 版本为 v25，与 Next.js 14 存在兼容性问题（`abort: Lazy deopt after a fast API call`），改用 `node@22` 启动开发服务器。
- 补充 `.env` 中缺失的 `NEXTAUTH_SECRET`、`NEXTAUTH_URL`，后续陆续写入 `DEEPSEEK_API_KEY`、`STABILITY_API_KEY`、`OPENAI_API_KEY`。

新增 API 路由：
- `src/app/api/ai/optimize-prompt/route.ts`：反向提示词优化接口。接收用户中文描述和可选参考图，调用 AI 模型生成两套优化提示词，每套包含中文版和英文版。初始使用 DeepSeek（有图用 `deepseek-vl2`，无图用 `deepseek-chat`），后切换为 OpenAI `gpt-4o`。
- `src/app/api/ai/generate-image/route.ts`：图片生成接口。支持 `model` 参数，`stable-diffusion-ultra` 调用 Stability AI `v2beta/stable-image/generate/ultra`，`gpt-image-2` 调用 OpenAI `gpt-image-1`（修复了初始误传 `response_format` 参数导致 400 错误的问题）。生成图片以 base64 解码后保存至 `public/uploads/`，返回本地路径。

首页改造（`src/app/(dashboard)/page.tsx`）：
- 将原静态 `<p>` 占位文字替换为真实 `<textarea>`，支持中文输入。
- "图片" 按钮接入隐藏 `<input type="file">`，上传后调用 `/api/upload`，显示缩略图预览。
- "生成" 按钮触发完整链路：① 调用 `optimize-prompt` 优化提示词 → ② 并行调用 `generate-image` 各生成一张图片。
- 新增生成进度状态：`optimizing`（优化提示词中）→ `generating`（生成图片中）→ `done` / `error`。
- 新增提示词展示区域（位于输入框与图片结果之间），展示两套方案的中文描述和英文提示词，优化步骤完成后即刻可见，无需等待图片生成完成。
- 新增生成结果区域，每张图片附下载按钮和"在 Studio 编辑"跳转。
- 新增模型选择 UI：两个卡片式切换按钮（Stable Image Ultra / GPT Image 2），选中状态高亮。
- 副标题、输入框占位文字、快捷标签全部改为家居家纺语境。

System Prompt 迭代：
- 第一版（DeepSeek）：通用商品场景摄影方向，七要素结构化输出。
- 第二版（切换 OpenAI gpt-4o）：增加家居家纺专项要求，负面限制，格式改为 `【方案X·中文优化提示词】` + `【方案X·English Prompt】` + `【方案X·Negative Prompt】`。
- 最终版（用户深度定制）：要求提示词不得字段罗列，须写成连续自然的可视化设计描述；细化产品用途、图案主题、设计风格、构图方式（含疏密/留白/拼接要求）、色彩方案（分风格细化水彩/简约/儿童/轻奢）、画面要求六大维度；负面提示词固定输出英文标准列表。

**验证**

- 每次改动后确认 `✓ Compiled` 无报错。
- 实测首页可正常输入文字、上传参考图；点击生成后提示词优化和图片生成链路跑通。
- 排查并修复了 Stability AI 404（mock 接口返回不存在的文件路径）和 OpenAI 400（多传 `response_format` 参数）两个接口错误。

**问题**

- DeepSeek `deepseek-vl2` 模型在 api.deepseek.com 商业接口的可用性待验证，目前已切换至 OpenAI gpt-4o，不再依赖 DeepSeek。
- 开发服务器需通过 `export PATH="/opt/homebrew/opt/node@22/bin:$PATH" && npm run dev` 启动，尚未固化到项目配置中。

**下一步**

- 可考虑将 Negative Prompt 从提示词展示区域透出给用户，并在调用 Stability AI 时传入 `negative_prompt` 字段以提升图片质量。
- Studio 页面 AI 面板（`src/components/canvas/AIPanel.tsx`）目前仍使用旧的 mock 流程，可按相同方式接入 `optimize-prompt` + `generate-image` 两个新接口。
- 模型选项后续可扩展（如 FLUX、Midjourney 等），当前架构已预留 `model` 参数。
