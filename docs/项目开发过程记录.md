# 项目开发过程记录

本文档用于持续记录本项目从当前雏形到完善上线的开发、优化和决策过程。新 agent 接手时应优先阅读本文件、`AGENTS.md` 和 `.codex/project-context/`。

## 当前接手摘要

- 当前项目是 AI 驱动的电商/营销视觉资产制作 SaaS。
- 技术栈：Next.js 14 App Router、React 18、TypeScript、Prisma、NextAuth、Fabric.js、Zustand、Tailwind CSS。
- 当前处于基础可维护性修复完成、Dashboard 顶部导航和仪表盘背景视觉调整阶段。
- 近期用户明确要求：小 UI 调整后不需要自动跑 `lint/build`，由用户本地检查。除非用户明确要求验证，后续 agent 不要主动运行检查命令。

### 3. 仪表盘背景素材层

已创建素材和组件目录：

- `public/dashboard/`
- `src/components/dashboard/`

当前已有图片：

- `public/dashboard/background-poster.png`

新增背景组件：

- `src/components/dashboard/DashboardBackground.tsx`

当前背景实现要求和状态：

- 图片作为 Dashboard 区域的固定背景层。
- 背景铺满顶部导航栏以下的整个视口。
- 背景从 `top-16` 开始，避开顶部导航栏。
- 背景使用 `fixed`，不会随着滚轮滚动。
- 页面内容作为上层滚动层显示在背景之上。
- 背景层 `pointer-events-none`，不会阻挡交互。
- 已移除半透明白色遮罩，让图片直接清晰展示。

背景接入位置：

- 在 `src/app/(dashboard)/layout.tsx` 的 `main` 内放置 `<DashboardBackground />`。
- 内容区域使用 `relative z-10`，并设置为独立滚动容器。

注意：用户对背景的最终期望是“红框区域全部由图片铺满，内容滚动但背景不滚动”。如果后续视觉仍不符合，需要优先检查 `layout.tsx` 中 `main` 和内容滚动容器的高度、overflow、z-index。
