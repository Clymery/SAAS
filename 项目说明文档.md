# SAAS 项目说明文档

本文面向刚拉取项目的新成员，说明如何从 0 到 1 运行项目、理解开发工具链与整体架构，并按主流全栈前端项目结构梳理当前目录。

## 一、使用流程：从 0 到 1 运行项目

### 1. 准备本地环境

建议本地准备以下工具：

- Node.js 18.17+，推荐 Node.js 20 LTS。
- npm，项目当前使用 `package-lock.json`，优先使用 npm。
- Git。
- VS Code 或 WebStorm 等 TypeScript 友好的编辑器。

项目使用 SQLite 作为本地数据库，不需要单独安装 MySQL 或 PostgreSQL。仓库里有 `prisma/dev.db`，新环境也可以通过 Prisma 迁移重新生成数据库结构。

### 2. 拉取代码并安装依赖

```bash
git clone <项目仓库地址>
cd SAAS
npm install
```

如果希望严格按锁文件安装依赖，可使用：

```bash
npm ci
```

### 3. 配置环境变量

项目根目录需要 `.env` 或 `.env.local`。当前本地开发可以参考下面配置：

```env
DATABASE_URL="file:./dev.db"
NEXTAUTH_URL="http://localhost:3000"
NEXTAUTH_SECRET="请替换为本地随机字符串"
REDIS_URL="redis://localhost:6379"
REPLICATE_API_TOKEN=""
```

说明：

- `DATABASE_URL` 指向 Prisma 的 SQLite 数据库。相对路径以 `prisma/schema.prisma` 所在目录为基准，因此 `file:./dev.db` 对应 `prisma/dev.db`。
- `NEXTAUTH_URL` 是 NextAuth 回调地址，本地默认是 `http://localhost:3000`。
- `NEXTAUTH_SECRET` 用于加密登录会话，生产环境必须替换。
- `REDIS_URL` 当前依赖中已有 Redis/BullMQ，但现有业务主要使用 mock AI 生成接口，Redis 不是本地启动的强依赖。
- `REPLICATE_API_TOKEN` 预留给真实 AI 生成服务，当前可以留空。

### 4. 初始化 Prisma Client 和数据库

首次运行建议执行：

```bash
npx prisma generate
npx prisma migrate dev
```

如果本地已经有 `prisma/dev.db` 且迁移状态正常，`migrate dev` 会复用现有 SQLite 数据库。数据库模型位于 `prisma/schema.prisma`，迁移文件位于 `prisma/migrations/`。

仓库包含 `prisma/seed.ts` 用于初始化模板数据，但当前 `package.json` 尚未配置 Prisma seed 命令，也未内置 `tsx`/`ts-node` 运行器。若后续需要稳定执行 seed，建议补充如下工具和脚本：

```bash
npm install -D tsx
```

并在 `package.json` 中增加：

```json
{
  "prisma": {
    "seed": "tsx prisma/seed.ts"
  }
}
```

之后可执行：

```bash
npx prisma db seed
```

### 5. 启动开发服务

```bash
npm run dev
```

浏览器打开：

```text
http://localhost:3000
```

### 6. 注册并进入系统

首次打开项目后，先访问注册页创建账号：

```text
http://localhost:3000/register
```

注册成功后会创建用户和对应的使用统计记录。之后通过登录页进入系统：

```text
http://localhost:3000/login
```

登录后可访问：

- `/`：仪表盘，展示项目数、素材数、生成额度等信息。
- `/projects`：项目管理，支持新建、搜索、复制、软删除项目。
- `/assets`：素材库，支持图片上传、预览、筛选。
- `/studio/[id]`：画布编辑器，包含工具栏、图层面板、属性面板、AI 面板、批量面板、模板弹窗。
- `/settings`：账号设置页。

### 7. 常用开发命令

```bash
npm run dev      # 启动本地开发服务
npm run build    # 构建生产版本
npm run start    # 启动生产构建
npm run lint     # 运行 Next.js ESLint
npx prisma studio # 打开 Prisma 数据库可视化工具
```

补充说明：当前 `next.config.mjs` 中设置了 `typescript.ignoreBuildErrors: true`，这意味着构建阶段会忽略 TypeScript 类型错误。新人接手时仍建议在 IDE 或单独类型检查中尽早处理类型问题。

## 二、开发工具链、项目框架与架构

### 1. 核心技术栈

本项目是一个基于 Next.js App Router 的全栈 Web 应用，主要技术如下：

| 分类 | 技术 | 作用 |
| --- | --- | --- |
| 应用框架 | Next.js 14 | 前端页面、服务端路由、API Route、构建与开发服务 |
| UI 框架 | React 18 | 组件化渲染 |
| 语言 | TypeScript | 类型约束与编辑器智能提示 |
| 样式 | Tailwind CSS | 原子化 CSS 样式 |
| UI 组件 | shadcn 风格组件、Base UI、lucide-react | 表单、弹窗、菜单、图标等基础组件 |
| 登录认证 | NextAuth v4 Credentials Provider | 邮箱密码登录、JWT Session |
| 数据访问 | Prisma ORM | 数据模型、迁移、SQLite 访问 |
| 数据库 | SQLite | 本地轻量数据库，文件位于 `prisma/dev.db` |
| 画布编辑 | Fabric.js | 图片、文本、形状等画布对象编辑 |
| 状态管理 | Zustand | 管理画布实例、选中对象、图层、撤销重做历史 |
| 文件上传 | Next.js API Route + `fs/promises` | 上传图片到 `public/uploads` |
| 消息提示 | sonner | Toast 通知能力 |
| 工具函数 | clsx、tailwind-merge、class-variance-authority | className 合并和组件变体 |

### 2. 整体架构

项目采用典型的 Next.js App Router 单仓全栈结构：

```text
浏览器页面 / React Client Component
        ↓ fetch / NextAuth
Next.js App Router 页面与 API Route
        ↓ Prisma Client
SQLite 数据库 prisma/dev.db
        ↓
public/uploads 本地静态资源
```

主要分层如下：

- 页面层：`src/app`，负责路由、布局和页面组合。
- 组件层：`src/components`，负责可复用 UI、共享组件、画布编辑器组件。
- 状态层：`src/stores`，当前主要是 `canvasStore.ts`，管理 Fabric 画布状态。
- 服务/工具层：`src/lib`，封装认证配置、Prisma Client、画布工具函数、通用 className 工具。
- 类型层：`src/types`，放业务类型、NextAuth 类型扩展、Fabric 类型补充。
- 数据层：`prisma`，定义数据库模型、迁移和本地 SQLite 数据库。
- 静态资源层：`public`，放公开访问的图片、上传文件和模板资源。

### 3. 路由设计

项目使用 App Router 的目录路由：

- `src/app/layout.tsx`：全局根布局，注入全局样式和 `Providers`。
- `src/app/(auth)/layout.tsx`：认证页布局。
- `src/app/(auth)/login/page.tsx`：登录页。
- `src/app/(auth)/register/page.tsx`：注册页。
- `src/app/(dashboard)/layout.tsx`：登录后的主布局，包含顶部导航、用户菜单、移动端菜单和鉴权跳转。
- `src/app/(dashboard)/page.tsx`：仪表盘首页。
- `src/app/(dashboard)/projects/page.tsx`：项目列表页。
- `src/app/(dashboard)/assets/page.tsx`：素材库页面。
- `src/app/(dashboard)/settings/page.tsx`：设置页。
- `src/app/(studio)/studio/[id]/page.tsx`：设计工作台页面，`(studio)` 是路由组，不影响访问路径。
- `src/app/api/**/route.ts`：服务端 API。

### 4. API 与业务模块

当前 API Route 主要包括：

- `api/auth/[...nextauth]`：NextAuth 认证入口。
- `api/register`：用户注册，使用 `bcryptjs` 加密密码，并创建 `UsageStat`。
- `api/projects`：项目列表、新建、更新、软删除。
- `api/projects/[id]`：单项目查询、更新、删除。
- `api/assets`：素材列表和素材记录创建。
- `api/upload`：接收上传文件，写入 `public/uploads`，返回静态访问路径。
- `api/templates`：模板列表。
- `api/generations`：AI 生成任务列表与创建。
- `api/generations/[id]`：单个生成任务查询。
- `api/ai/mock-generate`：模拟 AI 生成流程，将任务状态更新为 completed。
- `api/usage`：当前用户使用统计。

### 5. 数据模型设计

`prisma/schema.prisma` 中定义了 6 个核心模型：

- `User`：用户表，包含邮箱、姓名、头像、密码哈希、角色。
- `Project`：设计项目，保存项目名、描述、封面、画布 JSON、画布尺寸和状态。
- `Asset`：素材资源，保存图片、背景、模板、贴纸等资源路径。
- `Generation`：AI 生成任务，保存类型、状态、提示词、结果地址、错误信息和元数据。
- `Template`：系统模板，保存分类、场景、缩略图、画布尺寸和默认画布数据。
- `UsageStat`：用户使用统计，保存生成次数、图片数、存储占用和月度额度。

用户与项目、素材、生成任务之间是一对多关系；用户与使用统计是一对一关系；项目可关联多个生成任务。

### 6. 画布编辑器设计

设计工作台位于 `src/app/(studio)/studio/[id]/page.tsx`，由多个画布组件组合：

- `FabricCanvas`：初始化 Fabric.js 画布，监听对象新增、修改、删除和选区变化。
- `Toolbar`：提供画布操作入口，如添加元素、打开 AI 面板、打开模板等。
- `LayerPanel`：展示和管理画布图层。
- `PropertiesPanel`：编辑选中对象的属性。
- `AIPanel`：创建场景生成、智能抠图任务，并轮询生成结果。
- `BatchPanel`：批量处理相关面板。
- `TemplateModal`：模板选择弹窗。

画布全局状态由 `src/stores/canvasStore.ts` 管理，包括：

- 当前 Fabric Canvas 实例。
- 当前选中的 Fabric 对象。
- 图层列表。
- 撤销、重做历史。
- 序列化后的画布数据。

### 7. 认证与权限

认证配置位于 `src/lib/auth.ts`：

- 使用 NextAuth Credentials Provider。
- 登录时通过邮箱查询用户。
- 使用 `bcrypt.compare` 校验密码。
- Session 使用 JWT 策略。
- JWT 和 Session 回调中写入 `user.id` 与 `user.role`。

需要登录访问的 API 会通过 `getServerSession(authOptions)` 获取当前用户，未登录时返回 `401 Unauthorized`。

### 8. 当前项目注意事项

- 部分源码中文文案存在乱码，建议后续统一以 UTF-8 重新整理页面文案和 seed 数据。
- `public/uploads` 是本地上传目录，适合开发环境；生产环境建议切换到对象存储，如 S3、OSS、COS。
- `api/ai/mock-generate` 是模拟生成接口，当前不调用真实 AI 服务。
- 依赖中存在 `bullmq`、`ioredis`，但当前业务未形成完整后台队列 worker。
- `api/assets` 当前没有真正的 DELETE 接口，素材页删除只是在前端列表中移除，刷新后数据库记录仍会回来。
- `prisma/seed.ts` 当前中文内容也存在编码问题，并且 package 未配置 seed 运行方式。

推荐新人阅读顺序：

1. 先看 `package.json`，了解可执行脚本和核心依赖。
2. 再看 `prisma/schema.prisma`，理解业务数据模型。
3. 然后看 `src/app/(dashboard)/layout.tsx` 和 `src/app/(dashboard)/page.tsx`，理解登录后页面组织。
4. 接着看 `src/app/api/projects/route.ts`、`src/app/api/assets/route.ts`、`src/app/api/generations/route.ts`，理解前后端数据交互方式。
5. 最后看 `src/app/(studio)/studio/[id]/page.tsx`、`src/components/canvas/*`、`src/stores/canvasStore.ts`，理解画布编辑器的核心实现。
